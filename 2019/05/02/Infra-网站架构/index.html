    Flask


    Flask 不是我接触的第一个网站框架，但是是我用的最多的一个。原因有两点

    • 轻量。搭建一个可运行的网站基本只需要两个文件，几十行代码。对于搭建自己使用的小型服务，或是创业时候的可行性验证都是很好的选择。
    • 可拓展性强。在基础架构简单的情况下可以支持多种功能和稳定性上的优化，足以支持生产环境中的中小型服务。这一点之后还会细说。

    Data Model w/ MongoDB


    在数据库的选择上我也是倾向于选择轻量级的 NoSQL 数据库 MongoDB 作为敏捷开发的首选。当然 MongoDB因为他较差的安全性和可拓展性饱受诟病，但是我在这里还是如实的把我在开发创业项目时的设计和经历都写下来了。

    MongoDB 便于敏捷开发的一大原因是因为它是一个 schema-less 的数据库。数据的组织方式为 db, collection 和 document，分别对应SQL数据库中的 db, table 和 row。所谓的 schema-less指的是MongoDB的document不像SQL中的row一样有固定的schema，而是一个JSON-like的组织方式。也就是说在query MongoDB的时候会返回一个dict（用pymongo），而程序对于这个dict中有哪些key并没有一个系统上的认识。这样做的好处是数据的读取和写入及其方便，不需要任何ORM；坏处就是在之后的迭代开发中需要对数据进行人工的migration，很容易出现读写时的inconsistency，严重的时候甚至会有data corruption。
