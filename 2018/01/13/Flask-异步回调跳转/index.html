    • Flask 异步回调跳转的实现

    • 最近写创业项目的时候遇到了一个问题，应该也是比较的普遍：在线支付功能中如果使用了二维码，如何在用户扫码支付成功之后，令页面自动跳转？一般在线支付的 API（微信或支付宝）都会提供一个所谓的“回调链接”，也就是说用户扫码支付成功后，（e.g）微信的后台会给这个链接发去一个请求，里面会提供这次订单的信息（之类的）。仔细一想，实际上这个问题就是如何在客户端没有任何操作的情况下刷新页面。那么解决方法有如下两种：

      1. 用 Socket.io。我对 JS-based 网站架构不是特别熟，也不太清楚 Socket.io 是不是可以在非 Node 架构下使用，但是鉴于这个功能的本质就是在客户端用 JS 和服务器交互，所以我猜可以。以后有机会会把这部分补全（
      2. 用 block 的请求来模拟异步过程。这个就有点危险了，毕竟是异步系统嘛。不过很有意思，实现起来比较简单（只要想清楚了），不需要 import 其他的东西。具体来讲就是把这个过程拆分成几个请求：
        • makeQRCode: 根据订单信息产生支付二维码，将回调链接提供给支付机构，同时在 Runtime 中加上由订单号 identify 的锁。
        • notify: 回调链接，得到某个订单号支付成功的信息后解锁。
        • isPaid: 阻塞请求，又前端用 JS 访问，之后阻塞在订单号对应的锁上，直到锁解开。
