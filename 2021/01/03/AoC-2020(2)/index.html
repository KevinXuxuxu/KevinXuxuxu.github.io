    第一部分

    Day 10: Adapter Array *

    这道题也有点东西。
    第一问很简单，把所有数排序，然后统计一下顺序的所有值得差值中有几个1，几个3就好了。其实是下一问的铺垫。
    第二问要求给定的所有数能组成多少个不同的单调序列，使得其中没有相邻的数的差值超过3。简单的来讲就是看每一段连续的1差值能组合成多少种含有2和3的差值序列。
    e.g. 差值序列 (1, 1, 1, 1) 可以组成以下的7种差值序列

    考虑表达式 F(n) 代表 n 个1差值可以组成的不同的满足条件的差值序列的数量，有如下递归关系：F(n) = F(n-1) + F(n-2) + F(n-3), 既把1差值加在 n-1 个1差值的所有可能解上，把2差值加在 n-2 个1差值的所有可能解上，把3差值加在 n-3 个1差值的所有可能解上。
    通过类似求斐波那契数列的类似方法求出所需的 F 的值，然后把每一段连续的1差值所能组成的可能性相乘就可以得到答案。
    （不知是不是有意为之，在输入的过程中没有给2差值。如果有的话这一问会更复杂。）

    Day 11: Seating System *

    这是一道比较经典的模拟题，基本上是 Conway’s Game of Life 的翻版。第一问是经典的像素迭代由周围的八连通像素的状态决定，第二问是由八个方向第一个看到的像素状态决定，区别不是很大，就是稍微麻烦一点点。

    Day 12: Rain Risk *

    又是一道模拟题，要注意理解题意：第一问是直接的移动，第二问相当于有一个目标点（way point)，除了 F 以外的操作都是移动目标点而不是船本身。
    另外可以考虑一下怎么在代码上比较高效的实现旋转。

    Day 13: Shuttle Search *

    这道题很有意思。首先我们先确定输入的规模：假设序列中一共出现 n 个巴士，每个巴士的 id 最大为 m。

    • 第一问比较简单，可以从给定的时间点顺序向后查找，每次检查是不是被某个出现的巴士 id 整除。这个算法的复杂度是 O(nm'log(m)^2)，其中 m' 为巴士 id 之中的最小值，log(m)^2 为求余的复杂度（长除法），整体上还在多项式复杂度以内。

    • 第二问即：

      给定两组 n 个最大为 m 的正整数 b_i，a_i, 求最小的正整数 A 使得对于任意 i 有 A + a_i 被 b_i 整除

      很明显是一个数论中的同余问题，但是一时间没有思路（甚至 n = 2 的情况都没有思路）。比较暴力的方法是检查所有 b 的最大公约数以内的最大的 b_i 对于 b_i - a_i 的所有同余类，但是这样的复杂度是 O(m^n*log(m)^2)，肯定是不可接受的。
      最后只能查看同余的维基页面（emmm），果然发现了一个用中国剩余定理解线性同余方程组的方法。简单的来说线性同余方程组在任意一对 b 互质的情况下必有无数组解（在同余意义下是同一个解），可以通过求模逆元的方法构造出来。求模逆元的方法中比较容易理解和计算的方法是用扩展欧几里得算法。具体的证明可以看维基，应该不是很难懂。
      复杂度方面，根据维基页面欧几里得算法的复杂度应该是 O(log(m)^2), 所以整体的复杂度是 O(n^3*log(m)^2)，也在可以接受的多项式范围之内

    Day 14: Docking Data *

    这是一道挺麻烦的题，整体上需要注意一下十进制和二进制表示之间的转换，根据用的语言不同可能会有一些坑。

    • 第一问比较简单，可以吧 mask 里的 0 和 1 分别处理成两个二进制数，其中 mask_0 取所有 mask 里面所有的 0 取出放在对应的位置上，其他的位置用 1 填充；mask_1 同理。之后可以用按位取或（|）和按位取与（&）的方法来快速计算 mask 的结果。
    • 第二问比较难，可以先分析一下复杂度：给定 mask 中会有 m 个可变位（X），一共有 n 个写入操作，那么简单搜索的复杂度是 O(n*2^m)，这在 m很大的情况下是一个指数复杂度。辛运的是对于输入数据的统计得到了 m 在所有 mask 中的最大值只有 9，这个复杂度也就变得可以接受了。还有一个常数级别的优化可以做，就是从后往前来处理输入，这样的话先来的输入不会被覆盖，这样可以只记录哪些地址已经被写入，和所有被写入的值的和即可，不需要记录每一个值了。
    • 最后还是一个 BFS，用 (current_digit, partial_sum) 来定义搜索的状态会比较省空间。
